# 2023级数据库系统课程设计任务书

## 设计目的

数据库系统课程设计是在学生系统地学习了《数据库系统原理》课程后，按照关系型数据库的基本原理，综合运用所学的知识，设计开发一个小型的数据库管理信息系统。通过对一个实际问题的分析、设计与实现，将原理与应用相结合，使学生学会如何把书本上学到的知识用于解决实际问题，培养学生的动手能力；另一方面，使学生能深入理解和灵活掌握教学内容。

## 设计要求

1. 课程设计题目自行填充，独立完成；
2. 课程设计时间为1周；
3. 数据库管理系统不限（Oracle、SQL Server、MySQL、SQLite、MongoDB、达梦等），开发语言不限（C++、Java、C#、Python等），避免把学习语言的时间放在设计期间；
4. 从开始的系统需求分析到最后的实现，都要有详细的计划，设计文档应按照数据库设计的要求书写；
5. 系统中的数据表设计应合理、高效，尽量减少数据冗余，软件界面友好、健壮性强。

## 题目

> **####管理信息系统、####网站、####手机APP、####微信小程序**

### 1. 设计目的

学生根据所学的数据库系统原理与程序设计的知识，能够针对一个数据库管理信息系统，进行系统的需求分析，系统设计，数据库设计，编码，测试等，完成题目要求的功能，从而达到掌握开发一个**可视化**的小型数据库管理信息系统的目的。

### 2. 设计内容

> 可以是基于Web的网站或基于移动平台的手机APP、微信小程序、游戏软件等，**鼓励创新！**
>
> 题目可根据自己熟悉的事物填充和适当变更，例如：社交化读书笔记共享社区、AI出题与共享题库系统、自动化阅卷及统分系统、学生科研项目与成果数字化管理系统、社区健身设施预约与推荐系统、校园活动一体化管理平台等，**但不建议**仓库管理系统、图书管理信息系统、学生信息管理系统、酒店管理系统、药品进销存系统等。
>
> 要求实现数据的录入、删除、修改、查询等基本功能，**并且进行数据处理，得出有价值的结果。不能仅限于"数据怎么进去就怎么出来"，要对数据进行统计分析、加工处理。**
>
> 提供实体完整性、参照完整性、用户定义完整性等检查，**并在课程设计报告中明确描述。**
>
> 可根据自己对不同事物主要的管理需求进行扩充，例如数据图表展示、报表输出等功能。
>
> 聚焦用户友好的界面设计与人性化功能实现，可融入AI元素。

## 成绩评定

1. 由指导教师根据检查学生程序的情况、课程设计报告的质量和课程设计过程中的工作态度等综合打分。
2. 设计程序的检查由指导教师当面在计算机上检查测试，并同时对程序运行中的问题至少提出三个问题，学生当面回答，教师根据以上检查做好记载。
3. 独立按时完成规定的工作任务，不得弄虚作假。**严禁抄袭、复制设计报告内容，否则成绩以不及格计。发现课程设计报告基本雷同，一律不及格。**

## 课程设计报告的编写格式规范

### 题目

### 一、需求分析

介绍设计系统的**意义或用途**，以无歧义的陈述说明设计的任务，需要实现的功能，达到的要求、目标等。

画出系统功能模块图，并进行功能说明，用1、2、3、4......编号。

1. 功能1：
2. 功能2：
3. 功能3：
4. 功能4：
......

### 二、概念结构设计

画出E-R图。要求使用专门的制图软件，例如Visio。

### 三、逻辑结构设计

说明本系统中所用到的表结构及其联系，注明主码，外码等。例如：

**表1 合同操作流程表**

| 属性名   | 含义       | 类型     | 说明                     |
|----------|------------|----------|--------------------------|
| id       | id         | int      | 主键，每次自增1          |
| con_id   | 合同_id    | int      | 外键，引用合同表id       |
| type     | 操作类型   | int      | 1 会签，2 审批，3签订    |
| state    | 操作状态   | int      | 0 未完成，1 已完成，2 已否决 |
| use_id   | 用户_id    | int      | 外键，引用用户表id       |
| content  | 操作内容   | text     |                          |
| time     | 操作时间   | datetime |                          |
| del      | 删除状态   | int      | 0 未删除，1 已删除       |

或采取其他方式说明表结构，能清晰表达表结构及其联系为宜。

提醒：课堂所讲S-C模式都包含了3张表，你所设计的系统想必不会更简单咯？加油！

### 四、开发工具

列出软件的开发环境、开发工具、编程语言，如果用到了服务器或云平台，列出运行支撑环境，标明使用软件的版本号。

### 五、具体实现

截取系统运行的界面图，给出每一步的操作说明和输出结果、查询结果、报表等，解释所实现的实体完整性、参照完整性、用户定义完整性约束功能。

附上**核心**的程序代码，注意代码的易读性，提供必要的注释，复杂的代码需详细说明算法思路，可重点描述对数据的加工处理。

### 六、总结

针对前面设计实现的功能，对本系统的优缺点、有待完善的地方进行总结。
